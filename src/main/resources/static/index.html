Booking / Reservation POC

Three patterns in one Spring Boot app — pick a tab.

1. Seat hold (TTL) 2. Calendar conflict 3. Overbooking

What's inside

  • Seat hold — soft hold in memory (TTL 5 min) then hard hold in DB before payment. Scheduled sweeper auto-releases stale holds. Pattern works for hotel rooms or flight seats.
  • Calendar conflict — Calendly-style. Recurring availability rules + per-date exceptions, multi-timezone, buffer minutes, overlap detected with PESSIMISTIC_WRITE so the second booker loses cleanly.
  • Overbooking — Airline pattern. Sell beyond capacity by a factor, then bump volunteers (high willingness, cheap fare) first, involuntary by reverse fare-rank if needed. Returns compensation totals.
